# MyTrainings
Этот проект собран из ноутбука `MyTrainings.ipynb`.

Основные изменения относительно Colab-версии:

- убрана зависимость от `google.colab.drive`;
- чтение идёт из локальных CSV-файлов в папке `data/`;
- сохранение идёт в локальные файлы проекта;
- логика разнесена по нескольким Python-файлам.

### `visualization.py`

Позволяет смотреть графики объёма и интенсивности по большим и маленьким группам мышц.

### `additionDeletion.py`

Бывшая следующая большая ячейка с:

- просмотром тренировок по датам;
- Редактирование `data/trainings.csv`: удаление\добавление строк.

### `newExercise.py`

- редактирование/просмотр `data/exercises.csv`;
- позволяет открыть существующее упражнение;
- редактирует `difficulty_coeff` и коэффициенты по мышцам.


### `exercise_window.py`

Открывает отдельное desktop-окно, без `ipywidgets`, для графиков по упражнениям.

В окне есть:

- галочки для всех упражнений из `exercises.csv` и `trainings.csv`;
- кнопки `All` / `None` для упражнений;
- галочки и `All` / `None` для метрик;
- ползунки начала и конца диапазона дат; даты отображаются в формате `DDMMYYYY`;
- график для каждой пары «выбранное упражнение × выбранная метрика»;
- исходные точки, smooth по окну ±3 календарных дня и средняя линия.

Дни без выбранного упражнения не добавляются в ряд как нули, поэтому линия больше не проваливается к нулю между тренировками.

Метрики:

- `Сырой объём` = `weight * reps * sets`;
- `Умный объём` = `weight * reps * sets * difficulty_coeff`;
- `e1RM лучшего сета за день` = максимум `e1rm` за день по упражнению.

### `exerciseProgress.py`

Открывает отдельное desktop-окно, без `ipywidgets`, для просмотра лучших подходов по выбранному упражнению.

Она:

- даёт выбрать упражнение через выпадающий список;
- показывает таблицу `Вес / Повторы / Дата / Подходы / e1RM`;
- скрывает доминированные результаты: если есть другой подход с весом не меньше и повторами не меньше, слабая строка не показывается;
- при равных повторах оставляет только подход с большим весом;
- ничего не изменяет в CSV.

Например, если есть `158 × 1`, то `153 × 1` скрывается. Если есть `90 × 8`, то `85 × 4` тоже скрывается, потому что второй результат лучше и по весу, и по повторам.

## Формула объёма по мышцам

Для каждого сета сначала считается `e1RM` по формуле Эпли:

```math
e1RM = weight \cdot \left(1 + \frac{reps}{30}\right)
```

Дальше считается score с учётом упражнения:

```math
score_{set} = e1RM \cdot difficulty\_coeff \cdot sets
```

Где:

- `difficulty_coeff` — коэффициент упражнения из `exercises.csv`
- `sets` — количество подходов

Потом score распределяется по мелким мышечным группам:

```math
score_{small\_muscle} = score_{set} \cdot muscle\_coef
```

Затем мелкие группы агрегируются в крупные через `muscle_groups.csv`.

## Что сохраняется после визуализации

Файл `outputs/daily_muscle_volume_big_and_small_4metrics.csv` содержит:

- `date`

- колонки `e1rm__small__...`
- колонки `tonnage__big__...`
- колонки `tonnage__small__...`
- колонки `rawton__big__...`
- колонки `rawton__small__...`

Для всех календарных дней между минимальной и максимальной датой строится полный диапазон: если в день нет тренировки, в таблице стоят нули.

## Web frontend

Добавлен единый сайт в чёрно-белом стиле. Он не использует `ipywidgets` и позволяет переключаться между основными функциями проекта через левое меню.

Файл запуска:

```bash
python web_app.py
```

После запуска открой в браузере:

```text
http://127.0.0.1:8000
```

Разделы сайта:

- `Обзор` — краткая статистика и последняя тренировка;
- `Тренировки` — просмотр дней, bulk-добавление подходов, удаление выбранных строк;
- `Упражнения` — добавление и редактирование упражнения, `difficulty_coeff`, коэффициентов мышц и суммы;
- `Лучшие подходы` — таблица недоминированных подходов по упражнению; упражнение переключается сразу при смене выпадающего списка;
- `Графики упражнений` — цветные графики сырого объёма, умного объёма и лучшего дневного e1RM по упражнениям;
- `Графики мышц` — цветные графики по большим и маленьким группам мышц.

В графиках сами исходные точки не соединяются с нулём в дни без тренировки. При этом `smooth` считается по календарю с учётом дней без выбранного упражнения/мышечной группы; радиус сглаживания можно менять в интерфейсе. Графики сделаны цветными, а даты отображаются в формате `дд.мм.гггг`.

В разделе `Тренировки` режим bodyweight и отдельная кнопка добавления одной строки убраны: все новые подходы добавляются через bulk. Для всех упражнений вес вводится как обычный вес снаряда. Bulk использует дату из формы добавления и после добавления сохраняет выбранное упражнение и текст bulk. Поддерживаемые bulk-форматы: `80x3`, `3x85x2`, `5x(60x5)`.


В левом верхнем углу используется подпись `NeuroUnitaz Develop`. Логотип лежит в `static/neuro_unitaz_photo.png`; его можно заменить своим фото с тем же именем файла.

Для сайта добавлена зависимость `flask` в `requirements.txt`.

### Как залить web frontend в новую ветку

Из корня твоего git-репозитория:

```bash
git status
git switch main
git pull
git switch -c feature/web-frontend
```

Дальше скопируй файлы из этого архива в репозиторий с заменой старых файлов. Затем проверь запуск:

```bash
python -m venv .venv
# Windows PowerShell:
.\.venv\Scripts\Activate.ps1
# macOS/Linux:
# source .venv/bin/activate

pip install -r requirements.txt
python web_app.py
```

Если сайт открывается на `http://127.0.0.1:8000`, коммить изменения:

```bash
git status
git add web_app.py web_logic.py templates static README.md requirements.txt
git commit -m "Add black-and-white web frontend"
git push -u origin feature/web-frontend
```

Если ветка уже создана раньше, вместо `git switch -c feature/web-frontend` используй:

```bash
git switch feature/web-frontend
```
